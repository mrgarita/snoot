## プロダクト概要
かつて Mac で人気を博したパズルゲーム **Snood** をアプリとして再現する
ブラウザゲーム「**Snoot**」。バブルシューター系のゲームプレイ（画面上部に
並んだキャラクターを撃ち、同種を 3 つ以上つなげて消す）を、スマホ・PC の
どちらのブラウザでも遊べる形で提供する。まず Snood がどんなゲームかを調査し、
その調査結果に基づいて実装する 2 段階で進める。

### 厳守すべき制約
- **著作権配慮**：Snood には著作権があるため、Snood で使われたゲーム上の
  画像は一切使わない。別の画像を自作するか、フリー素材を使用する
- BGM・SE もフリー素材からの使用を検討する
- 難易度は **Easy / Normal / Hard** の 3 レベルを選択できる仕様とする
- **スマホでも PC でも遊べる** 環境とする（レスポンシブ／タッチ・マウス両対応）
- タイトルは「Snoot」とする
- 新規の有料サービスは使わない（既存資源優先・ゼロコスト重視）

### 完了条件
1. Snood のゲーム内容（ルール・難易度・スコアリング等）の調査結果が
   ドキュメントとして残っている
2. Easy / Normal / Hard の 3 レベルがそれぞれ仕様どおりに動作する
3. PC ブラウザ（マウス操作）とスマホブラウザ（タッチ操作）の両方で
   手動テストにより一通りプレイできることを確認する
4. 本番環境（デプロイ先）で動作確認が取れている

## 使用言語
プロジェクト配下での生成テキストはすべて日本語で書く。
対象は以下を含む。

- ユーザーへのチャット返信、進捗報告、ターン末の要約
- `AskUserQuestion` の質問文・選択肢のラベルと説明

- コミットメッセージ、PR タイトル、PR 本文
- README、CLAUDE.md、設計メモ、その他のドキュメント類
- エラーメッセージやログ出力のうち、エンドユーザー向けに表示される文字列

例外として **英語のまま** とするのは次のみ（言語仕様・エコシステム慣習のため）。

- プログラミング言語のキーワード、標準ライブラリ／フレームワーク／パッケージの API 名
- 識別子（変数名・関数名・クラス名・ファイル名・ディレクトリ名など）
- 設定ファイルのキー名、環境変数名

## 確定済みの実装方針
- **ブラウザ完結型**（サーバサイド処理なしの静的 Web アプリ）
- 技術構成：**Vite + TypeScript + Canvas 2D**（フレームワークなし）。
  `vite.config.ts` で `base: "./"` とし、サブディレクトリ配置でも動くビルドにする
- キャラクター画像は使わず **Canvas でコード描画**（著作権配慮のオリジナルキャラ 7 種、
  `src/characters.ts`）。SE は Web Audio API によるコード生成（`src/audio.ts`）
- バージョンは `package.json` の `version` が正。ゲーム画面に常時表示し、
  step2 を反復するたびに上げる（step4 の要件）
- デプロイ先候補：XServer レンタルサーバ（本番）、GitHub Pages（ステージング）
  ※ユーザー保有の既存資源を優先する
- 各 step の備忘録は `docs/site/*.html` に置き、`docs/site/index.html` から参照する
- 備忘録は文字だけにせず、**スクリーンショットや図を併載する**（画像は
  `scripts/make-doc-images.mjs` で `docs/site/img/` に生成。Snood 本家の画像は
  著作権配慮のため使わない）
