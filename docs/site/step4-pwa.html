step4：PWA 化とアプリアイコン変更（v0.9.0）

Snoot 開発備忘録 — 2026-06-17 / ← 目次へ戻る

このページについて

スマホ実機でのテストプレイを楽にするため、Snoot を PWA（Progressive Web App） にした（フィードバック #9、v0.9.0）。ブラウザから「ホーム画面に追加」でき、アドレスバーの ないスタンドアロン（全画面）でネイティブアプリのように起動できる。あわせて、 これまでの黄色いアプリアイコンを、ゲームキャラ 「ネムリ：寝息の浮き沈み」 （紫）に変更した。

アプリアイコンを「ネムリ」に変更

  • 旧アイコン（黄色いキャラ「ヒナタ」）から、紫の「ネムリ」へ差し替え。
  • 著作権配慮の方針どおり Snood 本家の画像は使わず、ゲーム本体の描画コード drawSnoot()（src/characters.ts の type=4）をそのまま流用して アイコン画像を生成した（scripts/make-icons.mjs）。ゲーム内のキャラと完全に同じ意匠。
  • 背景はゲーム本体の背景に馴染む紺 #1b2a4a。マスカブル版（Android の丸／角丸 マスク用）はキャラを安全域に収めるため少し小さめに描いている。
  • ブラウザのタブに出る favicon も、同じネムリの意匠（インライン SVG）へ変更した。

PWA でできること

  • ホーム画面に追加：スマホのブラウザの共有メニューやインストール導線から追加。 ホーム画面に紫ネムリのアイコンが並ぶ。
  • スタンドアロン起動：アドレスバーなしの全画面で起動し、テストプレイ時の没入感が増す。
  • オフライン対応：一度開けば Service Worker がアプリ本体をキャッシュし、 電波が不安定でも起動できる。

更新は自動で反映される

今後 step3/step4 を反復してこまめに更新していくので、インストール済みアプリが次回 起動時に自動で最新版へ入れ替わる方式（vite-plugin-pwa の registerType: "autoUpdate"）を採用した。アプリストアのような入れ直しは不要。

段階起きること
ビルド時新しいアセット一覧を埋め込んだ Service Worker が毎回生成される（中身が変わる）
次回アクセス時ブラウザが新しい Service Worker を検知し、裏で新アセットを取得
次回起動時取得済みの最新版へ自動で切り替わる（右下のバージョン表示が上がる）

技術メモ：どこに置いても動く相対パス設計

Snoot は GitHub Pages のバージョン別ディレクトリ（play/<tag>/・play/latest/）や XServer のサブディレクトリなど、置き場所が一定しない。そこで PWA の各設定を すべて相対パスにして、どの階層に置いても動くようにした。

  • vite.config.ts の base: "./" を維持。
  • manifest の start_url: "." / scope: "./"、アイコンも相対指定。
  • Service Worker の登録も register("./sw.js", { scope: "./" }) と相対。 これにより各バージョンのディレクトリごとに独立した PWA として成立する。
  • display: standalone / テーマ色・背景色は紺 #1b2a4a。
