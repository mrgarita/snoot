対応結果のスクリーンショット

#1：難易度によるキャラサイズの統一（v0.2.0）

PC でも盤面の横幅は難易度によらず一定で、列数が増えるほどキャラが小さくなる（スマホと同じ見た目）。

#2：スマホの操作エリア拡大（v0.3.0）

赤い点線と画面下端近くの砲台の間が広いスワイプ空間になり、指がキャラ・砲台にかぶらない。

#6：レベルアップによる段階的難化（v0.7.0）

同じ難易度（Easy）でも、レベルが上がると 初期行数が増え（盤面がより埋まる）、 キャラの種類が増え、危険ゲージが満タンになるまでの発射数が減る（天井がより速く降りる）。 下のスクリーンショットは同じ Easy の Lv.1 と Lv.5 の開始直後で、盤面の埋まり方が違う。

レベルごとの難化（Easy を起点とした例。各値は上限・下限で頭打ち／底打ち）：

レベル初期行数危険ゲージ容量キャラ種類数

ルール＝レベル+1ごとに「初期行数 +1」「危険ゲージ容量 −1」「キャラ種類数 +1（2 レベルごと）」。 盤面の列数は見た目を難易度間で揃えるためレベルでは変えない。Normal / Hard も同じルールで、 起点の値が高いぶん早く上限・下限に達する。

補足：原作 Snood ではどうだったか。 原作の Classic モードは難易度（Child/Easy/Medium/Hard/Evil）を選ぶと盤面サイズ・ 初期行数・危険ゲージ容量が固定で、面を進めても難化しない（新しいランダム盤面が 出るだけ）。難易度が段階的に上がるのは、Child から 1 レベルずつ難易度ラダーを上っていく Journey モードの方だった。Snoot は 3 難易度（Easy/Normal/Hard）を選ぶ構成なので、 原作 Classic の「難化しない」をそのまま踏襲せず、選んだ難易度を起点に少しずつ難化させる 現代的な方式（上表）を採った。

#7：ハイスコア保存機能（v0.8.0）

難易度ごとに自分のベスト 5 が残るようにした。新記録を出すと名前を入力でき （前回名が既定で入る）、タイトルの「ハイスコア」ボタンといつでも見返せる。スコアは ブラウザの localStorage に保存するので、閉じても次回に残る。

保存仕様：

  • 難易度別に Easy / Normal / Hard それぞれ上位 5 件を保持（得点は難易度で スケールが違うため別管理）。各記録は 名前・スコア・到達レベル・日付を持つ
  • 保存先は localStorage（キー snoot.highscores.v1）。サーバを使わない ブラウザ完結・ゼロコスト方式。データが壊れていても安全に空表示へフォールバックする
  • 記録のタイミングは「同じプレイ（連続レベル）につき 1 回」。「次のレベルへ」で 続く間は記録せず、最終的なゲームオーバー、または途中でタイトルへ戻ったときに、 積み上げたスコアを 1 回だけ記録する（同一プレイの途中得点で表が埋まらない）
補足：原作 Snood のハイスコア。 原作は 難易度別のローカル・ベスト表（名前入力あり）に加えて、 SnoodWorld.com の オンライン世界ランキング（不正対策のスコア検証コード付き）に 提出できる二層構造だった。Snoot はゼロコスト・ブラウザ完結の方針のため、オンライン 世界ランキングは採用せず、ローカルの難易度別ベスト表だけを localStorage で再現した。
