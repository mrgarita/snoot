1. Snood の概要

Snood（スヌード）は、1996 年に David M. Dobson 氏が Apple Macintosh 向けに開発した パズルゲーム。いわゆる「バブルシューター（Puzzle Bobble / パズルボブル系）」の ゲーム性を持つが、玉の代わりに 「snood」と呼ばれる表情豊かな顔のキャラクター を撃つのが最大の特徴。シェアウェアとして配布され、1990 年代末〜2000 年代前半に Mac・PC で絶大な人気を博し、後に Game Boy Advance などにも移植された。

2. 基本ルール

  • 画面上部に六角格子状（互い違い配置）に snood が並んでいる
  • 画面下部中央の 発射台（キャノン） を左右に回転させ、ランダムに選ばれた snood を撃ち上げる
  • 撃った snood は直線に飛び、左右の壁では反射 する。既存の snood か天井に当たると、その位置の格子に張り付く
  • 着弾によって 同じ種類の snood が 3 つ以上つながる と、そのグループ全体が消滅する
  • 消滅の結果、天井とのつながりを失った snood は落下 して盤面から除去される（落下による除去は得点が高い）
  • 盤面の snood をすべて消すとレベルクリア。snood が 画面下端のラインを越えるとゲームオーバー（ライフ制の場合は 1 ライフ喪失）

3. 危険ゲージ（Danger Meter）— Snood 最大の独自要素

一般的なバブルシューターとの最大の違いがこの仕組み。

  • snood を 1 発撃つたびに危険ゲージが増加 する
  • ゲージが満タンになると 天井が 1 段降下 し、盤面全体が下端に近づく。ゲージはリセットされる
  • snood を消す・落とすとゲージが減少 する。一度に多く消す（特に落下で大量除去する）ほど効果が大きい
  • このため「3 つだけ消して満足する」プレイは危険で、落下を狙って一度に大量の snood を除去する 戦略性が生まれる

時間制限は基本的に存在せず（Time Attack 系モードを除く）、「考える時間は無限にあるが、 無駄撃ちは確実に死を近づける」という独特の緊張感が Snood の中毒性の核心とされる。

4. キャラクター（snood の種類）

通常の snood は 7 種類。色と形・表情で見分ける。

特殊ピースも存在する：

  • Numbskull（ドクロ）：発射されることがなく、何ともマッチしない。周囲を消して「落とす」ことでしか除去できない
  • Stone（石）：着弾時に隣接する snood を消す
  • Wildcard：どの種類ともマッチする
  • Rowbuilder：横一列を埋める

5. 難易度

ランダム生成盤面で遊ぶ標準モードには Child / Easy / Medium / Hard / Evil の 5 段階がある。難易度は次の要素で差が付けられている。

  • 盤面の広さ（snood の総数）と snood の表示サイズ
  • 初期配置の行数
  • 危険ゲージの容量（満タンまでの発射数）
  • ドクロ（Numbskull）の出現有無

判明している具体値（GBA 版攻略情報より）：

難易度盤面（幅×高さ）初期行数危険ゲージ容量
Medium17 × 148 行8 発
Hard14 × 148 行7 発
Evil17 × 149 行6 発

6. ゲームモード

  • Classic：難易度を選んでランダム生成盤面をクリアしていく基本モード
  • Journey：低難易度から高難易度へ 1 レベルずつ続けて挑戦するモード
  • Puzzle：あらかじめデザインされた 50 面の固定配置を解くモード
  • Time Attack / Armageddon：危険ゲージが時間経過で自動上昇するモード

レベルはどう進むか（Classic と Journey）（2026-06-16 追記）

原作で「レベル」がどう進むかはモードによって異なる。

  • Classic：最初に難易度（Child / Easy / Medium / Hard / Evil）を 1 つ選ぶと、 盤面サイズ・初期行数・危険ゲージ容量はその難易度で固定。盤面を全消しすると 「レベルが進む（the level advances）」が、次の面も同じ難易度のランダム新盤面で、 面を進めても難化はしない。snood が画面下端のラインを越えるとライフを 1 つ失い、 尽きるとゲームオーバー（ライフ制）。
  • Journey：Child から始まり、1 レベルごとに難易度を 1 段ずつ上げていく （Child→Easy→Medium→Hard→Evil）。負けるまで続く。 「レベルアップ＝難化」を体感できるのはこのモード。

つまり原作で“レベルが上がると難しくなる”のは Journey 系の仕組みであって、Classic は 難易度据え置き。（Snoot では、選んだ難易度を起点にレベルで段階的に難化させる現代的な 方式を採用した。詳細は step3 のフィードバック #6 を参照）

7. スコアリング

  • 消した snood の数に応じて得点が入る
  • マッチで消すより「落として」除去する方が高得点（一度に大量落下させるほどボーナスが大きい）
  • クリア時には残りの危険ゲージ等に応じたボーナスが入る

ハイスコアの扱い（2026-06-16 追記）

原作のハイスコアは「ローカル」と「オンライン」の二層構造だった。

  • 難易度別のベストスコア表：Child / Easy / Medium / Hard / Evil / Custom / Puzzle / Journey / Armageddon それぞれに別々のハイスコア一覧を持つ （得点は難易度・モードでスケールが違うため）。
  • 名前入力：ゲーム内にローカル保存され、新記録時にプレイヤー名（Snooder 名）を入力する。
  • オンライン世界ランキング：SnoodWorld.com（旧 snood.net）の World Score list へ 提出できる。不正対策としてスコア検証コード（Score Verification Code）が付き、 mulligan（撃ち直し）を使うと提出スコアにチーズ印が付く、といったディテールもあった。

（Snoot は「ブラウザ完結・ゼロコスト・オフライン」の方針のため、オンライン世界ランキングは 採用せず、ローカルの難易度別ベスト表だけを localStorage で再現した。詳細は step3 のフィードバック #7 を参照）

8. Snoot（本プロジェクト）への反映方針

著作権上の制約（project.txt）により、Snood のキャラクター画像・名称はそのまま使わず、 オリジナルの顔キャラクターをコード描画で自作 する。再現するのは 「ゲームメカニクス（六角格子・発射・マッチ 3・落下・危険ゲージ）」であり、 ビジュアルは Snoot 独自のものとする。
  • 難易度は project.txt の指定どおり Easy / Normal / Hard の 3 段階 とし、 Snood の Easy / Medium / Hard 相当のパラメータ（キャラ種類数・初期行数・危険ゲージ容量）で差を付ける
  • スマホ縦持ちでも遊べるよう、盤面の列数は原作（14〜17 列)より抑えめに調整する （原作の列数はスマホでは 1 個あたりが小さくなりすぎるため）
  • v0.1.0 では Classic 相当の 1 モードのみ実装。Journey / Puzzle / 特殊ピースは フィードバック（step3〜4）を見て検討する
  • 危険ゲージ・落下ボーナスの仕組みは Snood の中毒性の核心なので初版から必ず実装する
